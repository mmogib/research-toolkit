# Pre-Submission Checklists

## Title Checklist

### Length & Format
- [ ] Length: 10-16 words (acceptable: up to 20)
- [ ] No undefined acronyms (or only well-established ones)
- [ ] Running head prepared if required (≤50 characters for SIAM)
- [ ] Proper capitalization (title case or sentence case per venue)

### Content & Keywords
- [ ] Primary keywords in first 65 characters
- [ ] Main contribution/method identified
- [ ] Problem domain/application clear
- [ ] Avoids vague openers ("A Study of...", "On the...")
- [ ] Specific rather than generic

### Searchability
- [ ] Uses standard terminology (searchable in MathSciNet/zbMATH)
- [ ] Would appear in relevant Google Scholar alerts
- [ ] Key terms match how experts would search

### Final Check
- [ ] Read title aloud—does it flow naturally?
- [ ] Would a colleague understand the topic from title alone?
- [ ] Does title accurately reflect the paper's main contribution?

## Abstract Checklist

### Length & Format
- [ ] Within word limit for target venue
- [ ] Single paragraph (unless structured format required)
- [ ] No equation numbers or internal references
- [ ] No undefined abbreviations
- [ ] Citations minimal and written out (not numbered)

### Opening
- [ ] Leads with main contribution (not background)
- [ ] Avoids "In this paper, we..."
- [ ] First sentence captures what you do/prove

### Content
- [ ] All major contributions mentioned
- [ ] Results stated specifically (rates, bounds, percentages)
- [ ] No proof techniques or methodology details
- [ ] Quantitative experimental results included (if applicable)

### Standalone Quality
- [ ] Readable without accessing the paper
- [ ] Accessible to researchers in related areas
- [ ] Technical terms either standard or defined
- [ ] Significance/impact clear

### Keywords
- [ ] 5-8 keywords selected
- [ ] Keywords complement (not duplicate) title terms
- [ ] MSC codes identified (if required)
- [ ] Standard terminology used

### Final Check
- [ ] Read abstract aloud—does it flow?
- [ ] Does it accurately represent the paper?
- [ ] Would you want to read this paper based on abstract?
- [ ] Have a colleague outside your area read it

## Quick Self-Review Questions

### For Titles
1. Can someone identify the topic in 3 seconds?
2. Are the main keywords present?
3. Is it specific enough to be meaningful?
4. Is it broad enough to attract readers?
5. Does it avoid common weak patterns?

### For Abstracts
1. What is the main contribution? (Should be in first 2 sentences)
2. What specifically do you prove/show? (Should be stated precisely)
3. Why does it matter? (Should be clear by the end)
4. Could someone in a related field understand it?
5. Does it make the reader want to read more?

## Document-Specific Additions

### For Theses
- [ ] Scope appropriately reflects full thesis (not just one chapter)
- [ ] Multiple contributions mentioned (for PhD)
- [ ] Accessible to all committee members
- [ ] Follows department formatting requirements

### For Grant Proposals
- [ ] Within character/word limit (strictly enforced)
- [ ] Intellectual merit clear
- [ ] Broader impacts addressed (if required)
- [ ] Accessible to panel (not just specialists)
- [ ] Innovation emphasized

### For Conference Papers
- [ ] Extremely concise (often 100-200 words)
- [ ] Single main contribution highlighted
- [ ] Would attract attendees to your talk

## Common Last-Minute Fixes

### Title Fixes
| Problem | Quick Fix |
|---------|-----------|
| Too long | Remove adjectives, combine phrases |
| Too vague | Add specific method or result |
| Acronym-heavy | Expand or use descriptive terms |
| Weak opener | Remove "A Study of", "On the" |

### Abstract Fixes
| Problem | Quick Fix |
|---------|-----------|
| Over word limit | Cut background, combine sentences |
| Weak opening | Move contribution to first sentence |
| Too vague | Add specific rates/percentages |
| Proof details | Delete methodology sentences |
| Missing results | Add key theorem statement |
